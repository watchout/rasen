# Campaign Brief: AI案件 受注前リスク診断

## Campaign ID

2026-06-ai-risk-diagnosis

## Goal

30日間で「AI案件 受注前リスク診断」の有料β版を1〜2件受注する。

## Offer

300万円以上のAI案件を受注する前に、実装可否、見積漏れ、追加質問、スコープ外を48時間で整理する診断サービス。

## Target

- DX支援会社
- Web制作会社
- システム開発会社
- 補助金支援会社
- kintone / Google Workspace / Microsoft 365 導入会社

## Qualification

以下に当てはまる相手を優先する。

- 顧客からAI案件の相談を受けている
- 300万円以上の案件を提案中または見積前
- RAG、生成AI、AIエージェント、社内データ活用、API連携などを含む
- 技術構成・見積項目・スコープ設計に不安がある
- 提案前に第三者視点のレビューを求めている

## Lead Magnet

第一候補:

- AI案件 受注前チェックリスト

第二候補:

- AI案件 見積漏れチェックリスト
- AI案件リスクスコア診断フォーム

## Core Message

そのAI案件、受注して赤字化しませんか？

AI案件は、提案時の見積漏れ・期待値調整ミス・運用設計不足でトラブル化します。提案前48時間で、実装可否・見積項目・追加質問・スコープ外を整理します。

## Channels

- 既存知人への個別連絡
- 開発者・DX支援会社への直接DM
- GitHub / 開発発信

## Human Approval Points

以下は必ず人間が確認する。

- 価格提示
- DM送信
- 申し込み返信
- 契約条件
- 保証表現
- 実績表現
- 顧客名や事例の記載

## Risks

- 実績がない段階で成果を断定しない
- 診断サービスなので、実装結果そのものを保証しない
- 「保険」と断定しない
- 技術名を前面に出しすぎない
- AIに詳しくない相手にも伝わる言葉にする

## Success Condition

30日以内に有料β版1件以上、または有力な商談3件以上が発生する。

## Stop / Pivot Condition

30社に接触して、返信・相談・興味がほぼない場合は、対象者または訴求を変更する。
