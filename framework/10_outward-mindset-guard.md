# 10 Outward Mindset Guard

Outward Mindset Guard は、Rasen の各モジュールで作られる出力が、顧客・取引先・紹介者・協業先を道具化していないかを確認する横断チェックです。

この Guard は、単独で売上を作るモジュールではありません。
Product Kernel から Feedback Loop まで、すべての判断に差し込む品質保証レイヤーです。

## 目的

ホルモジ式の強いオファーを、誠実で長期信頼につながる形に保つ。

```text
強い訴求
  + 守れる約束
  + 相手の現実理解
  + 実行しやすい納品
  = IYASAKA式の売れるオファー
```

## 基本チェック

すべての営業・マーケティング・納品出力について、以下を確認します。

```text
1. この出力は、相手の現実を見ているか
2. この出力は、相手の成功確率を上げているか
3. この出力は、相手の不安を煽るだけになっていないか
4. この出力は、こちらの売上都合だけで作られていないか
5. この出力は、断るべき相手を断れる設計になっているか
```

## Module Gates

### Product Kernel Gate

- 顧客の痛みは具体的か
- 顧客が本当に欲しい結果を、こちらの都合で置き換えていないか
- 売れるから売るのではなく、相手が買うべき理由があるか
- 売らない方が誠実な対象外条件を定義しているか

### Market Lens Gate

- 顧客の予算だけでなく、社内事情・担当者責任・現場負荷を見ているか
- 競合や代替手段を不当に低く見せていないか
- 緊急性を人工的に作っていないか
- 顧客が買わない合理的な理由を残しているか

### Offer Compiler Gate

- Dream Outcome は顧客の本当の成果か
- Perceived Likelihood は実績・証拠・納品物に基づいているか
- Time Delay を短く見せすぎていないか
- Effort & Sacrifice を軽く見せるだけでなく、本当に減らしているか
- Risk Reversal は守れる範囲か
- Scarcity は本当の制約か

### Pitch Compressor Gate

- 短くしたことで誇大表現になっていないか
- 恐怖訴求が強すぎないか
- 相手を責める表現になっていないか
- 冷静に判断できる余白があるか

### Asset Factory Gate

- LP、投稿、DM、商談台本の訴求が過剰に煽っていないか
- 無料導線が釣り餌ではなく、実務で使える価値を持っているか
- CTA が相手の次の一手として自然か

### Lead Engine Gate

- DMや紹介依頼が、相手の時間を奪う行為だと認識しているか
- 相手にとって読む価値があるか
- 一斉送信の都合で相手の状況を無視していないか
- 返信がないことを相手の問題にしていないか

### Sales Pipeline Gate

- 商談相手を「分かっていない客」と見ていないか
- 価格の反論を相手の理解不足だけにしていないか
- 相手が社内で説明する材料を渡しているか
- 今売るより、診断・保留・紹介・対象外判断の方が誠実ではないか

### Delivery System Gate

- 納品物は顧客が次の行動を取れる形になっているか
- 納品不足をアップセルで埋めようとしていないか
- 顧客が自走できる部分まで囲い込もうとしていないか
- 次の商品は本当に顧客の次の課題に合っているか

### Feedback Loop Gate

- 失注理由を相手のせいにしていないか
- 顧客の言葉を都合よく解釈していないか
- 数字だけでなく、信頼・納品満足・紹介可能性を見ているか
- 次週の改善が相手の行動しやすさにつながっているか

## In-the-Box Warning Signs

以下が出たら、出力や判断を見直します。

- 相手は分かっていない、と思った
- 断られた理由を相手の理解不足だけにした
- 高く売ること自体が目的になった
- 保証や希少性を強く見せるために表現を盛った
- 相手の社内説明や現場負荷を確認しなかった
- 納品後の顧客行動より、次のアップセルを優先した

## Output Requirement

営業・マーケティング・納品に関わる主要出力では、必要に応じて以下の短い確認欄を追加します。

```md
## Outward Mindset Check
- Customer reality:
- Customer risk:
- Customer effort reduced:
- Not-target / do-not-sell condition:
- Claim / guarantee safety:
```

## Final Rule

```text
売れるかだけでなく、相手が買ってよかったと言えるかを確認する。
```
